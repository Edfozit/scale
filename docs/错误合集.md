# 错误合集

> **本文档收集项目开发过程中遇到的各类错误及解决方案。AI 开发时必须阅读本文件，避免在同一个问题上重复犯错。**
>
> 遇到新的错误并解决后，请将经验补充到本文档对应分类中。

---

## 1. Room 数据库相关

### 1.1 Room 包名错误

- **现象**：编译报错找不到 Room 相关类
- **原因**：使用了 `androidx.room.*` 而非 `androidx.room3.*`
- **解决**：所有 Room 导入必须使用 `androidx.room3.*` 包名
- **Maven 坐标**：`androidx.room3:room3-runtime` / `room3-compiler`

### 1.2 Schema 变更未更新版本号

- **现象**：应用启动时抛出 `IllegalStateException` 并崩溃
- **原因**：修改了 `WeightRecord` 实体字段但未更新 `AppDatabase` 的 `version` 号
- **解决**：修改实体类后，必须同步更新 `AppDatabase` 的 `version` 号
- **注意**：当前使用 `fallbackToDestructiveMigration()`，schema 变更会清空所有数据

### 1.3 Room 注解参数名省略

- **现象**：编译报错，KSP 注解处理器无法识别
- **原因**：Room 3 的注解参数名（如 `@Insert(onConflict=...)`、`@Entity(tableName=...)`）为框架预定义，不可省略或修改
- **解决**：所有 Room 注解的参数名必须显式写出

## 2. Fragment 相关

### 2.1 hide/show 模式下数据不刷新

- **现象**：新增体重记录后，切换到图表页再切回来，数据没有更新
- **原因**：Fragment hide/show 模式下，切换不会触发 `onResume()`，而 ViewModel 数据仅在 `init` 时加载一次
- **解决**：在 Fragment 中重写 `onHiddenChanged(hidden: Boolean)`，当 `hidden == false` 时主动调用 ViewModel 刷新数据
- **适用**：所有 Fragment（ChartFragment、RecordsFragment 等）
- **代码示例**：
```kotlin
override fun onHiddenChanged(hidden: Boolean) {
    super.onHiddenChanged(hidden)
    if (!hidden) {
        viewModel.loadData(viewModel.currentDays)
    }
}
```

## 3. UI / 布局相关

### 3.1 Material 3 紫色水波纹/阴影

- **现象**：点击 Fragment 或底部导航栏时出现紫色阴影
- **原因**：Material 3 默认的 Ripple 效果和活动指示器颜色
- **解决**（需多管齐下）：
  1. `colorControlHighlight` = `@android:color/transparent`（禁用 Ripple）
  2. `android:colorControlHighlight` = `@android:color/transparent`（兼容属性）
  3. `colorSecondaryContainer` = 设为白色（控制底部导航活动指示器）
  4. 以上修改必须在 `values/themes.xml` 和 `values-night/themes.xml` 中同步
- **教训**：不能只修改 `colorControlHighlight`，还需要处理 `colorSecondaryContainer`

### 3.2 折线图日期标签被裁切

- **现象**：折线图底部日期标签（如 06/12）不可见或不完整
- **原因**：三层问题叠加
  1. CardView 的 `cardCornerRadius` 导致底部圆角裁切
  2. 图表 View 的右侧 padding 不足，末尾日期渲染空间不够
  3. 统计文字未居中
- **解决**：
  1. 将图表卡片的 `cardCornerRadius` 设为 `0dp`
  2. 增大 `WeightLineChartView` 的 `paddingRight`（如 20f → 50f）
  3. 为父 LinearLayout 添加 `android:gravity="center"`，为子 LinearLayout 添加 `android:gravity="center_horizontal"`

### 3.3 统计卡片文字未居中

- **现象**：统计数据卡片内的文字偏左，未水平居中
- **原因**：只在父容器设 `gravity="center"` 不够，子 LinearLayout 也需要单独设置
- **解决**：父容器 `android:gravity="center"` + 每个子 LinearLayout `android:gravity="center_horizontal"`

### 3.4 累计变化为 0 时显示双负号

- **现象**：累计变化为 0 时显示 `-- -0.0 斤`
- **原因**：格式化逻辑未单独处理 delta == 0 的场景
- **解决**：在 `ChartFragment.kt` 中对 `totalDelta` 格式化逻辑增加 `delta == 0f` 判断，直接显示 `0.0 斤`

### 3.5 日历视图随内容滚动

- **现象**：日历视图不固定，随下方内容一起滚动
- **原因**：日历视图放在了 ScrollView 内部
- **解决**：日历视图必须固定在界面顶部，不随下方 NestedScrollView 滚动

## 4. 工具 / 环境相关

### 4.1 jlink.exe 不存在

- **现象**：构建时报 `jlink executable does not exist` 错误
- **原因**：JAVA_HOME 指向了 JRE 目录，而 `jlink.exe` 仅存在于 JDK 中
- **解决**：确保 JAVA_HOME 指向 JDK 路径（非 JRE）

### 4.2 PowerShell 对象属性访问

- **现象**：PowerShell 命令中属性值为空或输出异常
- **原因**：直接写 `.Property` 无法访问 .NET 对象属性
- **解决**：必须使用 `$($obj.Property)` 语法；字符串拼接用双引号 + 子表达式 `$()`

## 5. AI 操作相关

### 5.1 文本替换失败（目标不存在）

- **现象**：search_replace 执行失败，找不到目标文本
- **原因**：文件已被之前的操作修改过，目标字符串已不存在
- **解决**：替换前先用 grep 确认目标字符串确实存在于文件中；如果不存在，重新读取文件确认当前状态

### 5.2 文本替换匹配多处

- **现象**：search_replace 报错匹配到多处相同文本
- **原因**：`original_text` 不够具体，在文件中匹配了多行
- **解决**：扩展 `original_text` 内容，加入足够的前后上下文（如多行属性组合），确保精准定位单一位置

### 5.3 颜色值替换前未验证

- **现象**：反复替换颜色值但始终不生效
- **原因**：目标颜色值在之前的操作中已被修改，但 AI 仍按旧值操作
- **解决**：颜色替换前先 grep 确认目标值是否还存在，不存在则重新读取文件

## 6. App 图标相关

### 6.1 Adaptive Icon XML 干扰

- **现象**：设置了 PNG 图标但系统仍显示旧图标或空白
- **原因**：`mipmap-anydpi-v26/` 目录下的 `ic_launcher.xml` 等 adaptive-icon XML 文件优先级高于 PNG
- **解决**：如需直接使用 PNG 图标，必须删除 `mipmap-anydpi-v26/` 下的 adaptive-icon XML 文件

---

## 7. 通用经验总结

| 原则 | 说明 |
|------|------|
| **先验证后操作** | 修改文件前先读取确认当前状态 |
| **先 grep 后替换** | 确保目标文本存在再执行替换 |
| **light/dark 双改** | 主题相关修改必须同时修改两套 themes.xml |
| **最小化改动** | 只改必要的部分，不要顺手改不相关的代码 |
| **遇到问题先记录** | 解决问题后及时补充到本文档 |
