# Scale - 体重记录 App 项目总览

## 1. 项目背景

**Scale** 是一款 Android 原生体重记录应用，帮助用户每日记录体重（早晨/晚间）、饮食和运动情况，并以日历视图、折线图表、记录列表等多种形式展示历史数据，辅助用户追踪体重变化趋势。

- **包名**：`com.example.myapplication`
- **最低支持版本**：Android 7.0（minSdk 24）
- **目标版本**：Android 16（compileSdk / targetSdk 36）

## 3. 项目架构（粗略）

采用 **单模块 + MVVM** 架构，以 Fragment 为页面单元，通过 BottomNavigationView 切换。

```
┌─────────────────────────────────────────────┐
│                 MainActivity                │
│  (NavHost, FragmentContainer + BottomNav)   │
├──────────┬──────────┬───────────┬───────────┤
│ Calendar │  Chart   │  Records  │  Profile  │
│ Fragment │ Fragment │  Fragment │  Fragment │
│  (日历)  │  (图表)  │  (列表)   │   (我)    │
├──────────┴──────────┼───────────┴───────────┤
│   MainViewModel     │   ChartViewModel      │
│  (日历/记录/统计)    │  (图表数据/统计)       │
├─────────────────────┴───────────────────────┤
│              data 层                         │
│  AppDatabase → WeightRecordDao → WeightRecord│
│          (Room 3, SQLite)                    │
└─────────────────────────────────────────────┘
```

### 3.1 分层说明

| 层 | 目录 | 职责 |
|----|------|------|
| **UI 层** | `ui/` | Fragment + 自定义 View + ViewModel，负责界面渲染和用户交互 |
| **数据层** | `data/` | Room 数据库实体、DAO、Database 单例 |
| **宿主** | `MainActivity.kt` | Fragment hide/show 容器 + 底部导航切换 |

### 3.2 核心数据模型

```kotlin
@Entity(tableName = "weight_records")
data class WeightRecord(
    @PrimaryKey val date: String,         // "2026-06-03"
    val morningWeight: Float?,            // 早晨体重（斤）
    val eveningWeight: Float?,            // 晚间体重（斤）
    val foodNote: String?,                // 饮食记录
    val exerciseNote: String?,            // 运动记录
    val createdAt: Long
)
```

### 3.3 四个页面 Tab

| Tab | Fragment | 功能 |
|-----|----------|------|
| 记录 | `CalendarFragment` | 日历视图 + 当日体重录入 + 月度统计卡片 |
| 图表 | `ChartFragment` | 折线图 + 多时间范围统计 |
| 日历 | `RecordsFragment` | 体重记录滚动列表 |
| 我 | `ProfileFragment` | 个人信息/设置（待完善） |

## 4. 项目开发流程

### 4.1 环境准备

1. **JDK**：需要 JDK 17+（AGP 9.x 要求）
2. **Android SDK**：compileSdk 36
3. **Gradle**：使用项目内置 Gradle Wrapper（`gradlew.bat`），无需单独安装
4. **IDE**：推荐使用 Android Studio Ladybug+

### 4.2 构建 & 运行

```bash
# Windows
gradlew.bat assembleDebug      # 编译 Debug APK
gradlew.bat installDebug       # 编译并安装到连接设备
```

### 4.3 开发迭代流程

```
需求分析 → 修改布局/资源 XML → 修改 Kotlin 代码 → 构建验证 → 功能测试
     ↑                                                        |
     └────────────── 反馈修复 ←─────────────────────────────────┘
```

关键开发步骤：

1. **数据层变更**：修改 `WeightRecord` 实体 → 更新 `AppDatabase` version 号 → 配置迁移策略（当前为 `fallbackToDestructiveMigration`，即 schema 变更时清空重建）
2. **UI 层变更**：修改 Fragment 布局 XML → 更新对应 Fragment/ViewModel 代码 → View Binding 自动生成
3. **新增页面**：创建 Fragment + 布局 XML → 在 `bottom_nav_menu.xml` 添加菜单项 → `MainActivity` 注册切换逻辑

### 4.4 重要约束与规范

| 约束 | 说明 |
|------|------|
| Room 包名 | 必须使用 `androidx.room3`（非 `androidx.room`） |
| Room 注解参数 | 注解参数名必须显式写出，如 `@PrimaryKey` |
| View Binding | 已开启，布局文件自动生成 Binding 类，不使用 `findViewById` |
| Fragment 切换 | 采用 hide/show 模式（非 Navigation Component），Fragment 实例通过 `lazy` 缓存 |
| 数据库迁移 | 当前使用 `fallbackToDestructiveMigration()`，schema 变更会清空数据 |
| Material 3 | 已禁用 Ripple 水波纹效果，底部导航栏活动指示器透明化 |
| 日历视图 | 固定在界面顶部，不随下方内容滚动 |
| 体重单位 | 斤（非公斤） |

| T1 | **Room 包名** | 必须使用 `androidx.room3`，**禁止**使用 `androidx.room`。Room 3 的 Maven 坐标为 `androidx.room3:room3-runtime` / `room3-compiler` |

| # | 规范 | 说明 |
|---|------|------|
| A1 | **实体类业务语义命名** | 实体类命名须体现业务含义（如 `WeightRecord`），**禁止**泛化技术名称（如 `Entity`、`Item`） |

| 记录行高 | `48dp` | 当日记录卡片每行高度 |
| FAB | `normal` 尺寸 | 右下角，`margin="24dp"` |

#### 8.3.3 日历视图专属

| 元素 | 尺寸/样式 | 说明 |
|------|----------|------|
| 月份导航栏 | 高 `44dp` | 左右箭头 `36dp × 36dp`，内边距 `6dp` |
| 日历控件 | `padding="6dp"` | WeightCalendarView 外边距 |
| 日期格子 | 白色圆角卡片 | 背景 `#FFFFFF` |
| 日期数字 | 普通字重 | **禁止加粗**，所有状态垂直位置一致 |

### 8.4 组件规范

#### 8.4.1 CardView 使用标准

```xml
<!-- 标准卡片（有阴影） -->
<androidx.cardview.widget.CardView
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:layout_marginHorizontal="12dp"
    app:cardCornerRadius="12dp"
    app:cardElevation="1dp">

<!-- 融合卡片（无阴影，背景色与页面一致） -->
<androidx.cardview.widget.CardView
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:layout_marginHorizontal="12dp"
    app:cardBackgroundColor="#F2F4F7"
    app:cardCornerRadius="12dp"
    app:cardElevation="0dp">
```

#### 8.4.2 输入框标准

- 使用 `TextInputLayout` + `TextInputEditText`（Material 3）
- `boxBackgroundMode="outline"` 描边模式
- 输入框间距：首个 `marginTop="16dp"`，后续 `marginTop="8dp"`
- 文字大小 `16sp`

#### 8.4.3 按钮标准

- 主按钮：`MaterialButton`（默认 filled 样式）
- 次按钮：`Widget.Material3.Button.TextButton`（文字按钮样式）
- 按钮布局：右对齐，主次间距 `8dp`
- 弹窗按钮区 `marginTop="16dp"`

#### 8.4.4 底部导航栏

- 使用 `BottomNavigationView`，自定义样式 `Widget.MyApp.BottomNavigationView`
- `labelVisibilityMode="labeled"`（始终显示标签）
- 图标/文字颜色通过 `bottom_nav_tint.xml` 选择器统一控制
- 活动指示器透明（`itemActiveIndicatorStyle` 设为透明）

### 8.5 主题配置

#### 8.5.1 主题继承链

```
Theme.Material3.DayNight.NoActionBar
  └── Base.Theme.MyApplication
       ├── colorControlHighlight = @android:color/transparent   （禁用 Ripple）
       ├── android:colorControlHighlight = @android:color/transparent
       ├── android:statusBarColor = @android:color/white
       ├── android:windowLightStatusBar = true
       └── Theme.MyApplication （最终使用主题）
```

#### 8.5.2 Light / Dark 同步规则

| 属性 | Light 主题 | Dark 主题 | 说明 |
|------|-----------|----------|------|
| `colorControlHighlight` | `transparent` | `transparent` | 双主题一致 |
| `android:colorControlHighlight` | `transparent` | `transparent` | 双主题一致 |
| `android:statusBarColor` | `white` | （默认） | 仅 Light 显式设置 |
| `android:windowLightStatusBar` | `true` | （默认） | 仅 Light 显式设置 |

**强制约束**：任何主题属性修改必须同时更新 `values/themes.xml` 和 `values-night/themes.xml`。
